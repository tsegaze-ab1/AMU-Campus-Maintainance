Campus Maintenance Request Management System
Entry-Level Software Engineering Project Portfolio

Project Summary
I developed this web-based Campus Maintenance Request Management System to solve a real operational problem in schools and universities: maintenance issues are often reported late, tracked manually, and resolved without visibility.

This project centralizes issue reporting, assignment, status tracking, and communication in one role-based platform. It transforms scattered reports into a clear digital workflow from request creation to resolution.

Real Campus Problems This System Solves
1. Unstructured issue reporting
Students can submit maintenance problems with category, location, priority, and description, replacing verbal or paper-based reporting.

2. Slow assignment and response
Admins can assign requests directly to technicians, reducing delays and confusion about ownership.

3. Poor visibility of work progress
All users can track request status (new, in progress, resolved) in real time.

4. Weak accountability and communication
Timeline comments and updates create a traceable record of actions, notes, and decisions.

5. Limited decision-making data
Dashboards provide KPI summaries and charts so teams can measure workload, progress, and completion rates.

How The Website Works (Sequential Flow)
Step 1: Public Access and Authentication
- Users enter through a public landing page.
- New users register an account.
- Existing users log in with secure credentials.

Step 2: Role-Based Access Routing
- After login, each user is redirected to a role-specific dashboard:
  student, technician, or admin.

Step 3: Student Request Creation
- Student opens Create Request.
- Student submits category, title, location, priority, and description.
- System validates input and stores the request as new.

Step 4: Admin Review and Assignment
- Admin views all requests in a centralized table.
- Admin checks details and assigns a technician.
- Admin can also update status and leave operational comments.

Step 5: Technician Execution
- Technician sees only assigned tasks.
- Technician updates status in controlled sequence:
  new -> in_progress -> resolved.
- Technician adds work notes and progress updates.

Step 6: Student Follow-Up and Transparency
- Student checks request details and current status.
- Student views assigned staff and update timeline.
- Student can add comments for clarification.

Step 7: Monitoring and Reporting
- Dashboards display live counts and status distribution.
- Teams use KPI cards and charts to monitor service performance.

UI/UX Design Highlights
1. Clear role-focused navigation
Each dashboard shows only the features relevant to that user role, reducing cognitive overload.

2. Responsive layout for desktop and mobile
Sidebar and off-canvas navigation support efficient use on both large and small screens.

3. Visual status language
Color-coded badges and KPI cards communicate pending, in-progress, and completed states instantly.

4. Action-oriented interface
Primary actions like Create Request, View, Update, and Assign are placed prominently to support fast execution.

5. Readable operational data
Tables, progress bars, and charts make workload and outcomes easy to scan and understand.

6. Continuous feedback
Users receive direct feedback through alerts, status labels, and timeline comments.

Dashboard Overview
Student Dashboard
- Tracks personal request counts and completion percentage.
- Shows recent requests with priority and status.
- Includes status distribution chart and quick links to create/view requests.

Technician Dashboard
- Displays assigned workload and completion metrics.
- Highlights pending, in-progress, and completed assignments.
- Provides quick access to request view/update actions.

Admin Dashboard
- Presents user distribution by role (students, technicians, admins).
- Shows overall request pipeline and completion percentage.
- Includes charts for request status and role distribution.
- Enables access to request management, category management, and user management.

Technical Scope
- PHP (server-side logic and role control)
- MySQL (users, categories, requests, comments, attachments schema)
- Bootstrap (responsive interface)
- Chart.js (dashboard analytics)
- CSRF protection and input validation in core request workflows
